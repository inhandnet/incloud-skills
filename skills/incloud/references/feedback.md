# InCloud Skill 反馈

当用户在使用 incloud skill 过程中发现问题（命令示例过时、评级标准不准、流程缺失、文档错误等），引导收集结构化反馈并通过平台 Feedback API 提交。

## 反馈收集流程

### 1. 从会话上下文预提取

先回顾当前会话，自动提取：
- 涉及的 skill 或 reference 文件（如 `signal-analysis.md`、`diagnostics.md`）
- AI 给出的错误建议或不准确的输出
- 用户执行过的相关命令和实际结果

### 2. 引导用户补充

逐步确认以下信息（已从上下文提取到的直接展示让用户确认，无需重复问）：

1. **哪里不对**：具体是哪个文档/哪段内容有问题
2. **实际发生了什么**：AI 给出了什么错误的建议或结果
3. **期望行为**：正确的做法应该是什么

### 3. 组装反馈内容

将收集的信息组装为一段结构化文本（content 字段，上限 1000 字）：

```
[文件名] 问题概述

问题：{用户描述的问题}
涉及文件：{skill 或 reference 文件名}
实际行为：{AI 给出的错误建议}
期望行为：{用户认为正确的做法}
上下文：{关键的会话片段}
```

内容要精炼，优先保留问题描述和期望行为，会话上下文只摘关键片段。

### 4. 提交

向用户展示组装好的内容，确认后通过 `incloud api` 提交：

```bash
incloud api POST /api/v1/feedbacks --payload '{
  "app": "incloud-cli",
  "type": "ISSUE",
  "content": "<组装好的反馈内容>"
}'
```

`type` 根据内容选择：
- `ISSUE` — 文档错误、命令示例不对
- `SUGGESTION` — 流程改进建议、缺失内容

## 注意事项

- 反馈内容用中文，命令和技术术语保持原样
- 不要在反馈中包含用户的认证信息、token 或敏感数据
- content 上限 1000 字，超长时精简会话上下文部分
